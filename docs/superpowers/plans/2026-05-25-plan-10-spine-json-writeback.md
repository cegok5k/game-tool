# Plan 10 — Spine JSON Write-Back Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When the editor dispatches an `UPDATE_TRANSFORM` event for a spine-bone, persist the change to the corresponding native Spine JSON file (`media/skeletons_json/<...>/Skeleton.json`) on disk, so edits survive iframe reloads and feed the existing gulp pipeline.

**Architecture:**
1. A pure `patchBone(skeletonJson, boneName, partial)` function returns a new JSON with the bone's transform fields patched. TDD'd against a real fixture from big-bait's `main_scene/main_scene/Skeleton.json`.
2. A `resolveSkeletonFile(skeletonId)` helper converts the studio's dotted skeleton id (e.g. `"main_scene.main_scene.Skeleton"`) into a project-relative file path (e.g. `"media/skeletons_json/main_scene/main_scene/Skeleton.json"`), per the convention documented in `D:\work\big-bait\client\CLAUDE.md`.
3. A `spinePatchStore` (Zustand) accumulates pending patches keyed by file path, so a flurry of gizmo drags batches into a single disk write.
4. `CanvasPanel` enqueues patches on `UPDATE_TRANSFORM`, debounces, then writes via `PlatformAdapter.fs.writeFile`. Console panel gets a "Saved" log entry on success.

**Why not the ROADMAP's original target (`compiled/game/media/skeletons/...`):** those files are regenerated by gulp on every build — edits there get blown away. Investigation against `D:\work\big-bait` confirmed the editable source for bone transforms lives in `media/skeletons_json/`. Native Spine 4.2 JSON format, bones with `name, parent, x?, y?, rotation?, scaleX?, scaleY?, shearX?, shearY?, length?, inherit?`.

**Out of scope (Plan 11):** Bridge integration with the real big-bait game. Big-bait does not currently load our bridge SDK; transforming the gizmo→file loop against the real game requires injecting the bridge into the studio's `jst/jst-spine` runtime. Plan 10 validates via the existing test-game bridge using big-bait's real `Skeleton.json` as the TDD fixture, so when bridge injection lands the write-back machinery already works against real data shapes.

**Tech Stack:** TypeScript 5.7 (strict, `verbatimModuleSyntax`), Zustand 5, Vitest 2, existing `PlatformAdapter`. No new dependencies.

// src/spine/resolveSkeletonFile.ts
const ROOT = 'media/skeletons_json'

export function resolveSkeletonFile(skeletonId: string): string {
  if (!skeletonId) {
    throw new Error('resolveSkeletonFile: empty skeleton id')
  }
  const parts = skeletonId.split('.')
  if (parts.length < 2) {
    throw new Error(`resolveSkeletonFile: id must contain at least one dot, got "${skeletonId}"`)
  }
  const skeletonName = parts[parts.length - 1]
  const dirs = parts.slice(0, -1)
  return `${ROOT}/${dirs.join('/')}/${skeletonName}.json`
}
```

// src/stores/spinePatchStore.ts
import { create } from 'zustand'
import type { BonePatch } from '../spine/spineJsonTypes'

type State = {
  pending: ReadonlyMap<string, ReadonlyMap<string, BonePatch>>
  enqueue: (skeletonFile: string, boneName: string, patch: BonePatch) => void
  clearFile: (skeletonFile: string) => void
  pendingFiles: () => string[]
  reset: () => void
}

export const useSpinePatchStore = create<State>((set, get) => ({
  pending: new Map(),

  enqueue: (file, bone, patch) => {
    const next = new Map(get().pending as Map<string, Map<string, BonePatch>>)
    const inner = new Map(next.get(file) ?? new Map<string, BonePatch>())
    inner.set(bone, { ...inner.get(bone), ...patch })
    next.set(file, inner)
    set({ pending: next })
  },

  clearFile: (file) => {
    const next = new Map(get().pending as Map<string, Map<string, BonePatch>>)
    next.delete(file)
    set({ pending: next })
  },

  pendingFiles: () => Array.from(get().pending.keys()),

  reset: () => set({ pending: new Map() }),
}))
```

// src/spine/applyPatchBatch.ts
import { patchBone } from './patchBone'
import type { BonePatch, SpineSkeletonJson } from './spineJsonTypes'

function detectIndent(text: string): string | number {
  // Find the first indented line; prefer tab vs space.
  const match = text.match(/\n([ \t]+)/)
  if (!match) return 2
  const indent = match[1]
  if (indent.startsWith('\t')) return '\t'
  return indent.length
}

export function applyPatchBatch(
  originalText: string,
  patches: ReadonlyMap<string, BonePatch>,
): string {
  const indent = detectIndent(originalText)
  let json = JSON.parse(originalText) as SpineSkeletonJson
  for (const [boneName, patch] of patches) {
    json = patchBone(json, boneName, patch)
  }
  return JSON.stringify(json, null, indent) + (originalText.endsWith('\n') ? '\n' : '')

```ts
let flushTimer: ReturnType<typeof setTimeout> | null = null
const FLUSH_DEBOUNCE_MS = 300

function scheduleFlush(platform: PlatformAdapter, handle: FolderHandle): void {
  if (flushTimer !== null) clearTimeout(flushTimer)
  flushTimer = setTimeout(() => {
    flushTimer = null
    void flushPendingPatches(platform, handle)
  }, FLUSH_DEBOUNCE_MS)
}

async function flushPendingPatches(platform: PlatformAdapter, handle: FolderHandle): Promise<void> {
  const store = useSpinePatchStore.getState()
  const console = useConsoleStore.getState()
  for (const file of store.pendingFiles()) {
    const bonePatches = store.pending.get(file)!
    try {
      const original = await platform.fs.readText(handle, file)
      const next = applyPatchBatch(original, bonePatches)
      await platform.fs.writeFile(handle, file, new TextEncoder().encode(next))
      store.clearFile(file)
      console.log('info', `Saved ${file} (${bonePatches.size} bone${bonePatches.size === 1 ? '' : 's'})`)
    } catch (err) {
      const message = err instanceof Error ? err.message : String(err)
      console.log('error', `Failed to save ${file}: ${message}`)
    }
  }
}
```

Pull `platform` from `getPlatform()` and `handle` from `projectStore`. If `projectStore.handle` is null, skip the flush.
